Quiero que construyas una aplicación web tipo SPA (Single Page Application) para la gestión de PQRSF (Peticiones, Quejas, Reclamos, Sugerencias y Felicitaciones) como una app satélite externa que se integra con un sistema principal llamado “SGE - Sistema de Gestión Educativa”.

OBJETIVO ARQUITECTÓNICO

La aplicación PQRSF debe ser un cliente desacoplado que consuma la API del sistema principal (SGE), sin tener acceso directo a la base de datos.

Debe cumplir con:

Arquitectura desacoplada (cliente de API)
Consumo exclusivo de endpoints REST (/api/v1/)
No duplicar lógica de negocio ni datos
Escalable para futuras integraciones (móvil, otros portales)

CONTEXTO DEL SISTEMA PRINCIPAL (SGE)

Backend: Django + Django REST Framework
Base de datos: PostgreSQL
API centralizada: /api/v1/
Autenticación: Microsoft Entra ID (Azure AD) y/o sistema interno
Seguridad: API Keys con scopes (basico, salud, financiero, etc.)
Documentación: Swagger
Integraciones: SharePoint (gestión documental)

El SGE ya gestiona:

Usuarios (estudiantes, empleados, administrativos)
Dependencias
Información académica y administrativa

ARQUITECTURA DE LA APP PQRSF

La app debe ser:

Frontend:

Vite
JavaScript o React
Tailwind CSS
Arquitectura modular por dominios

Backend:

No crear backend propio inicialmente
Consumir directamente la API del SGE

CONSUMO DE API

La aplicación debe consumir endpoints como:

POST /api/v1/pqrsf/solicitudes/
GET /api/v1/pqrsf/solicitudes/
GET /api/v1/pqrsf/solicitudes/{id}/
PATCH /api/v1/pqrsf/solicitudes/{id}/estado/
GET /api/v1/pqrsf/dashboard/

También endpoints del core:

GET /api/v1/usuarios/
GET /api/v1/dependencias/

AUTENTICACIÓN

La app debe soportar:

Login mediante Microsoft Entra ID (si aplica)
Uso de tokens JWT o API Keys según el tipo de usuario
Manejo de sesiones en frontend

ROLES Y ACCESO

La aplicación debe manejar roles:

Usuario (crea solicitudes)
Dependencia (gestiona solicitudes)
Administrador

Cada rol debe ver únicamente la información que le corresponde.
